## 프롬프트 작성 시 주의사항 및 회고
- 여전히 답변을 마크다운 형식으로 출력해주지 않음. 따라서 해당 프롬프트를 따로 사용.
```
너가 작성해준 자료야.
내용을 잘 파악하고 나서 마크다운 형식으로 내용 변경 없이 그대로 출력해줘.

[자료]
chat gpt 내용 그대로 스크래핑
```
- AI 활용을 위하여 작성할 것, 생각할 것, 활용 자료 첨부로 프롬프트 작성.

## 작성 프롬프트
```
해당 문제 상황에 대해서 페르소나, 배경, 핵심 문제,
해결 과제를 보고 너 스스로 [해결 과제]를 풀어봐.

내가 주는 자료 [AI 프롬프트 전략]을 활용해서 [작성 내용]을 작성해주면 돼.

[해결 과제]
가상 문제 상황
1. 가상 문제 상황
- 페르소나: 광고 대행사의 콘텐츠 제작 팀장

- 배경: 당신은 클라이언트 'A'(신규 뷰티 브랜드)의 신제품 런칭을 위해 3가지 버전의 유튜브 범퍼 광고(6초) 영상을 제작하여 집행했습니다. 3가지 영상은 각각 다른 소구점(A: 성분 강조, B: 사용 후기, C: 할인 혜택)을 강조했습니다.

- 핵심 문제: 캠페인 2주차 데이터 분석 결과, A와 B 영상은 평균 클릭률(CTR) 2.5%, **구매 전환율 1.8%**를 기록하며 순항 중입니다. 하지만 **가장 많은 예산을 투입한 C 영상(할인 혜택 강조)의 CTR은 0.8%**에 불과하며, **구매 전환율은 0.1%**로 처참한 수준입니다. 특히 C 영상은 3초 이내 이탈률이 80%에 육박합니다.

- 해결 과제: 클라이언트는 C 영상의 즉각적인 중단과 예산 환불까지 고려하고 있습니다. 당신은 C 영상의 성과가 저조한 구체적인 원인을 분석하고, 이 영상을 즉시 수정(re-edit)하거나 대체할 수 있는 긴급 대응 방안을 내일 오전 회의까지 보고해야 합니다.

2. 훈련 초점 (힌트)
이 문제는 [원인 분석] 능력을 평가

[작성 내용]
1. 프롬프트 유형과 선택 이유
2. 작성 위해 추가 필요 요건
3. 문제 해결 프롬프트 
4. 프롬프트 실행 결과

[AI 프롬프트 전략]
제공 자료 스크래핑
```

## 프롬프트 작성 결과
# 1. 프롬프트 유형과 선택 이유

**역할 부여(Role-Playing)** — 광고 제작 팀장의 관점에서 실무적 판단·우선순위 도출 필요 (전문성 확보).  
**CoT(Chain-of-Thought)** — 원인 분석에서 단계적 추론을 유도하여 뿌리원인(creative/UX/타깃/노출) 도출.  
**제약 조건(Constraint-Based)** — 6초 범퍼 규격(0–6s), 텍스트 비율, 브랜드 가이드(톤·금지어) 등 엄격 준수 필요.  
**예시 기반(Few-Shot)** — 성공 사례(A/B 영상 구조 예시)를 제공해 동일한 패턴으로 재생성.  
**조건부 생성(Conditional)** — 타깃(신규 고객/리타겟/관심고객)별 맞춤 버전 생성.  

이 조합은 분석의 깊이(CoT)와 실무 적용성(역할부여·제약)을 동시에 확보합니다.

# 2. 작성 위해 추가 필요 요건 (즉시 수집 권장)

(회의 전까지 가능한 범위에서 준비)

- C 영상의 전체 6초 영상 파일(원본), 편집 타임라인(컷별 초 단위), 스크립트 자막 원문  
- 노출 채널(유튜브 인스트림/유튜브 Shorts/네트워크), 타깃 세그먼트(연령, 성별, 관심사)  
- 3초·6초 지점의 시청자 리텐션 그래프(초 단위), CTR 클릭 후 랜딩 URL, 랜딩 페이지 스크린샷(모바일)  
- 광고 소재별 노출 빈도·시간대·입찰 전략(예: 최대CPM 또는 tCPA)  
- 브랜드 금지어·색채·로고 사용 규칙, 프로모션 조건(기간/코드/최대 할인)  
- 기존 A/B(성분/사용후기)에서 잘 먹힌 메시지 키워드·CTA 문구  

회의 전 일부 항목(리텐션 그래프, 영상 파일)만 확보돼도 즉시 재편집안 실행 가능합니다.

# 3. 문제 해결 프롬프트 (AI에 투입할 실제 프롬프트들)

아래 프롬프트들을 AI 편집·카피·분석 툴(또는 내부 PM에게 복붙)로 즉시 실행하세요. (모두 한국어)

## A. 원인 분석용 (CoT + 역할부여)
> 당신은 디지털 광고 성과 분석가이자 광고 크리에이티브 디렉터입니다.  
> 아래 데이터를 바탕으로 'C_할인' 6초 범퍼 광고의 성과저조(CTR 0.8%, 전환율 0.1%, 3초 이탈률 80%)의 가능한 원인들을 단계별로 분석하고,  
> 각 원인별 증거(로그/지표로 확인할 항목)와 우선순위(긴급도·수정 난이도)를 표 형식으로 정리해 주세요.  
> 마지막에 '즉시 실행 가능한 3가지 개입안(재편집/대체)'을 적어 주세요.

---

## B. 재편집(6초) 스크립트 생성 (역할+제약)
> 당신은 광고 크리에이티브 디렉터입니다.  
> 아래 제약을 준수하여 6초짜리 범퍼 광고용 재편집 스크립트 3안을 제시해 주세요.  
> **제약:** 총 길이 6초, 첫 0.8초(초반프레임) 내 훅 필요, 자막은 2줄 이하, 브랜드 로고 엔드프레임 1초 보장, 모바일 세로/가로 공통 사용 가능 텍스트 포함.  
> **대상:** (1) 신규 고객(관심 유도) (2) 가격 민감 고객(즉시 클릭 유도) (3) 리타겟(장바구니 이탈자).  
> 각 안에 (a) 초단위 연출(0.0–6.0s), (b) 음성/자막 텍스트, (c) CTA 텍스트, (d) 편집 노트(컷/모션/사운드)를 서술형으로 작성.

---

## C. 랜딩/UX 체크리스트 (인스턴스 기반)
> C 영상의 CTR은 낮으나 클릭 후 전환도 낮습니다.  
> 당신은 UX전문가입니다.  
> 클릭 후 랜딩 페이지에서 구매전환을 높이기 위해 즉시 점검하고 수정해야 할 10가지 항목(우선순위 포함)을 모바일 우선 관점으로 작성해 주세요.

---

## D. 긴급 예산/집행 권고(제약+역할)
> 당신은 성과 마케터입니다.  
> 캠페인 2주차 상황에서 'C 영상'에 대해 다음 옵션을 평가·권고하세요:  
> (1) 즉시 중단·환불 요청 (2) 예산 축소(%) + 재편집 후 재투입 (3) 리타겟/맞춤음성으로 전환.  
> 각 옵션의 리스크·예상효과·권장 실행 조건을 3문장 이내 요약해 주세요.

# 4. 프롬프트 실행 결과  
(실제 실행을 가정한 실전 산출물 — 즉시 보고용)

---

## A. 원인 분석 요약 (우선순위 높은 원인 4가지)

1. **초기 훅 부재 (가장 결정적)**  
   - **증거:** 0–3초 이탈률 80% / 전체 CTR 저조 → 초반 메시지로 관심 못 끔.  
   - **설명:** 할인 메시지가 뒤에 나타나거나 텍스트·비주얼이 약해 초반 스크롤·스킵 유발.

2. **메시지-랜딩 불일치(Expectation gap)**  
   - **증거:** 클릭 후 전환 0.1% → 할인 문구 클릭 유도했지만 랜딩에서 동일 프로모션·코드 미노출 가능성.

3. **타깃·크리에이티브 미스매치**  
   - **증거:** A·B는 CTR 2.5%로 선호 메시지 패턴 존재. C는 할인 톤이 채널·타깃에 안맞음(브랜드 포지셔닝과 충돌).

4. **AD 프레임·CTA 약함(비주얼·사운드 문제)**  
   - **증거:** 첫 프레임 텍스트 과다/로고 우선 노출, 음성·사운드 효과 없음 → 임팩트 약함.

---

## B. 즉시 실행 가능한 긴급 대체안(우선순위 추천)

### 긴급안 1 (권장 우선 실행, 도입 시간 2–4시간)  
**‘강력훅’ 버전 (new-cut) — 신규 고객용**

- **0.0–0.5s:** 블랙→빨강(브랜드 색) 플래시 + 큰 텍스트 “지금만”(한 단어) — 훅  
- **0.5–2.0s:** 제품(클로즈업) + 텍스트 “첫구매 30%” (브랜드·할인 혼합)  
- **2.0–4.5s:** 짧은 사용 장면(1인칭 손 동작) + “장바구니에 담기” 느낌  
- **4.5–6.0s:** CTA “바로보기” + 로고(1초 고정)

**편집 노트:** 0.0–0.5초에 가장 강한 시각 자극(단어·색 대비).  
사운드: 두드리는 킥(0.0) + 클릭 소리(4.5s).  
**기대효과:** 초반 리텐션 개선 → CTR 상승(목표 1.8–2.0%).

---

### 긴급안 2 (가격민감 타깃, 즉시 클릭 유도)  
**‘숫자+한정’ 버전**

- **0.0–0.7s:** 큰 숫자 “30%” 팝업(애니메이션) + “한정” 빨간 딱지  
- **0.7–3.0s:** 할인 적용된 실제 '태그 가격' 노출(전/후 가격)  
- **3.0–5.0s:** “코드: NEW30” + 클릭 유도(버튼 모사)  
- **5.0–6.0s:** 로고+짧은 CTA(모바일 터치 안내)

**편집 노트:** 가격·코드가 랜딩에도 동일하게 노출되어야 함(랜딩 연계 필수).  
**기대효과:** 클릭 의도는 높이나 랜딩 불일치 시 전환 0 유지 가능 — 랜딩 체크 병행 필수.

---

### 긴급안 3 (리타겟 전용)  
**‘사회증거+빠른 리마인드’ 버전**

- **0.0–1.0s:** “고객이 말해요: 좋아요♥”(간단한 별·아이콘) — 사회증거 훅  
- **1.0–3.5s:** 실제 리뷰 한 줄(자막) + 제품 사용 컷 빠르게 노출  
- **3.5–5.0s:** 할인·코드 노출(짧게)  
- **5.0–6.0s:** CTA “다시보기” + 로고

**편집 노트:** 리타겟 풀(장바구니·사이트 방문자)에게만 노출.  
**기대효과:** 리타겟팅 효율 상승, 전환율 회복(목표 0.8–1.2%).

---

## C. 랜딩 페이지 즉시 점검(모바일 우선) — 10개 체크리스트(우선순위 상위 5개)

1. **할인 문구·코드 일치:** 광고의 할인%, 코드가 랜딩에도 동일한 위치에 노출되는지.  
2. **페이지 로드 속도(≤2s 권장):** 모바일 로드 지연 시 이탈 확률 급증.  
3. **원클릭 구매/간소 결제 버튼 노출:** CTA와 결제 흐름 단축.  
4. **프로모션 유효성(기간/재고) 명확화:** 혼동 방지.  
5. **노출된 제품 바로 연결:** 광고에서 본 SKU가 랜딩 최상단에 노출.  
(하위: 신뢰 배지, 반품정책, 카트 상태 동기화, 유입 UTM·파라미터 확인, 앱 딥링크 점검)

---

## D. 예산/집행 권고(즉시)

- **중단 대신:** 즉시 완전 중단은 비권장(환불 이슈·데이터 샘플이 아직 축적됨).  
- **권장 실행안:** C 영상 일부(50–70%) 예산 감축 → 나머지로 재편집 3안(A/B/C 리타겟 버전) 신속 집행(테스트 예산).  
- **조건:** 재편집 1번(or 2번) 재집행 시 랜딩 페이지 동기화(할인 텍스트·코드) 완료 후 재투입.  
- **대응기간 목표:** 재편집·QA(모바일) 2–4시간 내 완료 → 6시간 내 소규모 재집행 권장.

---

## E. A/B 테스트 계획(간단)

- **목표지표:** CTR 상승(목표 1.8% 이상), 전환율(구매) 0.8% 이상(단기 목표).  
- **분할:** 동일 타깃 풀 내에서 3개 크리에이티브 병렬(각각 25% 트래픽), 나머지 25%는 기존 A/B 영상 유지.  
- **평가기간:** 최소 2–3일(샘플 수 보장).  
- **중간점검:** 24시간 리텐션·CTR 확인.  
- **우선순위 지표:** 3초 리텐션 → CTR → 전환율 순.

# 회의용 1슬라이드(요약) — 회의에 바로 붙여넣기

**현상:** C(할인) CTR 0.8% / 전환 0.1% / 3s 이탈 80% → 중단 고민.  
**분석(요약):** 초반 훅 부재·메시지-랜딩 불일치·타깃 미스매치 3대 원인.  
**즉시 권고(우선순위):** 예산 50–70% 감축 → 3가지 재편집안 제작(신규/가격민감/리타겟) → 랜딩 동기화 후 소규모 재집행.  
**기대:** CTR 1.8% 수준 회복, 전환 0.8% 목표.  
**필요자료(회수):** 원본 영상·리텐션 그래프·랜딩 스크린샷(지금 즉시 확보해 주세요).

# 추가로 제공할 편집 리소스(붙여넣기용 카피)
