## 1️⃣ 프롬프트 유형과 선택 이유  
**[유형 조합: 역할 부여 + CoT + 조건부 생성 + 제약 조건]**

| 프롬프트 유형 | 활용 목적 | 적용 방식 |
|----------------|------------|------------|
| 🎭 **역할 부여(Role-Playing)** | 전략 컨설턴트 관점에서 전문적 사고 적용 | “당신은 TMT 전문 전략 컨설턴트입니다.”로 시작하여, 시장·데이터 기반 분석 중심의 답변 유도 |
| 🧠 **CoT (Chain-of-Thought)** | 문제 원인-결과-대응 단계를 체계적으로 도출 | “단계별로 사고하고, 각 단계의 논리적 근거를 명시하라”는 지시 포함 |
| ⚙️ **조건부 생성(Conditional)** | ‘Z-Play’의 사례와 시장 트렌드(숏폼, 인터랙티브, AI 큐레이션 등)를 반드시 반영 | “아래 조건(시장 트렌드, 경쟁사 분석)을 모두 포함하여 작성하라”로 구체화 |
| 📏 **제약 조건(Constraint-Based)** | 결과물 형식(전략 3가지 + 실행 로드맵) 명확화 | “전략 방향은 3가지로 한정, 각 전략은 ‘핵심 개념 / 실행 계획 / 기대 효과’ 구조로 제시” |

> 💡 **선택 이유:**  
> 전략적 판단이 필요한 복합 과제에서는 ‘전문적 사고(역할 부여)’ + ‘단계별 논리 전개(CoT)’ +  
> ‘트렌드 및 데이터 조건(조건부 생성)’을 결합해야 결과의 정확성과 실효성을 확보할 수 있음.
