## 프롬프트 작성 시 주의사항 및 회고
- 어제는 이튿날 작성했던 프롬프트를 바탕으로 똑같이 재구성 했으나 오늘은 프롬프트를 다시 작성해봄.
- 여전히 md 출력에 있어서 문법 오류가 생겨 일부 수정하는 불편함이 있었음.
- 단계별 작성과 '자세히'와 같은 추가 사항에 대해 요구하니 조금 더 잘 나오는 것 같음.
- 엊그제와 어제 prompt games에서는 역할부여를 하지 않았으나 추가 사항으로 넣어봄.
 - 해당 답변에 있어서 본인이 '전문가'가 되는 것이 아니라 '관점'에서 본다고 함. 다른 역할부여 프롬프트로 작성해봐야 할 필요성을 느낌.

## 작성 프롬프트(Chat gpt)
```
너는 현재 다방면(마케팅, 컨설팅)으로 분야를 넓혀 여러 시장을 확보하고 있는 IT 업계 30년 차 전문가야. 내가 해결해야 하는 사항을 자료를 통해 아주 자세하게 작성해줘.

내가 주어진 [가상 문제 상황]을 제공하는 자료 [technical_spec], [learn_promptengineerings]을 바탕으로 [해결 사항]에 맞춰 자세하게 잘 작성해주면 돼.

[가상 문제 상황]
가상 문제 상황 스크래핑.

[해결 사항]
1. 프롬프트 유형과 선택 이유
[learn_promptengineerings]을 주로 참고.

2. 작성을 위한 추가 필요 요건
모든 자료를 참고하여 너가 작성하기에 부족했던 데이터나 사항을 긴밀히 텍스트로 작성.

3. 문제 해결 프롬프트
IT 업계 30년 차 전문가로서 AI를 위한 프롬프트 작성을 아주 자세하게 작성해줘.

4. 프롬프트 실행 결과
모든 자료를 바탕으로 상세히 작성. 

[technical_spec]
Docs 파일 스크래핑

[learn_promptengineerings]
github 스크래핑
```

## 프롬프트 작성 결과
# 🧩 AI 업무 자동화 SaaS 숏폼 캠페인 기획서  
**작성자:** IT 업계 30년 차 전문가 (Creative Tech Consultant / Marketing Strategist)  
**목적:** 복잡한 기술 명세를 MZ세대 친화적 멀티미디어 스토리로 재해석하여,  
“기술의 매력”을 감각적으로 전달하는 30초 숏폼 영상 시나리오 제안

---

## 1️⃣ 프롬프트 유형과 선택 이유  
*(출처: [learn_promptengineerings])*

| 사용 프롬프트 유형 | 적용 목적 | 설명 |
|------------------|-------------|--------|
| 🎭 **역할 부여(Role-Playing)** | 전문성·현장감 확보 | "광고 대행사 크리에이티브 디렉터"의 관점에서 실제 기획처럼 구성 |
| 🧠 **CoT (Chain-of-Thought)** | 복잡한 기술 → 대중 감성 변환 | 기술 사양을 단계별로 해석 → 핵심 가치(‘시간 절약’, ‘쉬운 자동화’)로 단순화 |
| 🎨 **페르소나(Persona)** | MZ세대 감성 반영 | ‘디지털 라이프스타일에 익숙한 직장인’ 톤으로 감각적 표현 |
| ⚙️ **재구성(Restructuring)** | 기술 문서 → 콘텐츠 시나리오 변환 | 명세서 기반 기술 정보를 숏폼 영상 스크립트·카피로 전환 |
| 🎯 **제약 조건(Constraint)** | 영상 포맷 최적화 | “30초 이내 / 인스타그램·틱톡용 / 핵심 기능 1개씩 강조” 제한 부여 |

> ✅ **프롬프트 조합 구조 예시**
> - “당신은 MZ세대 감성에 강한 크리에이티브 디렉터입니다(역할 부여).  
>  아래 기술 명세의 핵심 기능을 단계별로 분석(CoT)하고,  
>  30초 이하 숏폼 광고 시나리오로 재구성(재구성)해 주세요.  
>  톤은 위트 있고 간결하게, MZ세대 직장인에게 매력적으로(페르소나+제약 조건).”

콘텐츠 제작 관점에서 아래 항목들이 추가로 필요합니다.

| 구분 | 부족한 데이터 | 필요 이유 |
|------|----------------|------------|
| 🎯 **타깃 감성 데이터** | MZ세대 직장인의 “업무 스트레스 / 자동화 욕구” 관련 인사이트 | ‘공감→전환’ 포인트 확보 |

| 🧭 **브랜드 톤앤매너 가이드** | 유머러스 / 스마트 / 프리미엄 등 브랜드 성격 명시 필요 | 영상 톤 조정 및 카피 정합성 |

| 📊 **경쟁사 벤치마킹 자료** | Zapier, Make.com 등 글로벌 SaaS 숏폼 사례 | 시각 언어의 방향성 참고 |

## 3️⃣ 문제 해결 프롬프트 (AI 명령어 설계)
🎬 **문제 해결 프롬프트 예시 (최적화 버전)**

```
당신은 광고 대행사의 크리에이티브 디렉터입니다.
주 타깃은 MZ세대 직장인이며, 복잡한 기술 명세를 직관적이고 감각적인 숏폼 영상으로 표현해야 합니다.
아래 AI 업무 자동화 SaaS의 기술 명세를 바탕으로, 30초 이내의 인스타그램 릴스/틱톡 광고 시나리오 3개를 제시하세요.

[출력 조건]

각 시나리오는 "핵심 카피 / 비주얼 컨셉 / BGM 아이디어"로 구성

기술적 기능을 MZ세대의 일상적 언어로 재해석

전달 톤은 ‘힙하고 스마트하며 공감 가능한’ 방식

AI 기반 자동화의 효용(시간 절약, 간편함, 연결성)을 스토리 중심으로 표현

[참고 키워드]

핵심 기능: 워크플로우 자동화, 자연어 기반 업무 처리, Slack·Notion·Gmail 통합

감정 포인트: ‘번아웃 탈출’, ‘퇴근 시간 단축’, ‘내 손이 아닌 AI가 대신 해주는 일상’
```

## 4️⃣ 프롬프트 실행 결과 (실제 시나리오 제안 3가지)

---

### 🎬 시나리오 ① : **“퇴근 5시의 반전”**
**핵심 카피:**  
> “AI가 내일 할 일을… 오늘 끝냈다.”

**비주얼 컨셉:**  
- [장면1] 시계 4:59 ⏰ — 직장인 얼굴 피곤  
- [장면2] “워크플로우 실행” 클릭 → Slack, Notion 자동으로 움직이는 화면  
- [장면3] 직장인이 웃으며 “퇴근합니다👋”  
- [장면4] 자막: “코딩 없이, 클릭 한 번으로 자동화.  
**AI가 퇴근을 앞당깁니다.**”

**BGM 아이디어:**  
- Phonk 계열 리듬감 있는 “Work smarter, not harder” 루프 사운드  
- 퇴근 전 긴장감→해방감으로 전환되는 템포 구성  

---

### 🎬 시나리오 ② : **“손 대신 AI가 일하는 아침”**
**핵심 카피:**  
> “AI가 출근을 먼저 했다 ☕”

**비주얼 컨셉:**  
- [장면1] 알람 울림 / 침대 속 직장인  
- [장면2] AI가 Gmail 정리, 보고서 작성, 메시지 응답 자동 처리  
- [장면3] 커피 마시며 휴대폰 확인 → “오늘 일정 완료됨 ✅”  
- [장면4] 자막: “AI가 당신의 출근 루틴을 자동화합니다.”

**BGM 아이디어:**  
- 로파이 + 하이퍼팝 믹스 (“Morning Flow”)  
- 출근 전 여유감과 자동화의 ‘스마트함’을 강조  

---

### 🎬 시나리오 ③ : **“슬랙이 나보다 빠르다”**
**핵심 카피:**  
> “Slack이 먼저 답장했다. 내가 보내기도 전에.”

**비주얼 컨셉:**  
- [장면1] Slack 알림 ‘고객 문의 답변 완료’  
- [장면2] “AI가 알아서 처리했습니다 🤖”  
- [장면3] 개발자/마케터/HR 등 다양한 직군 등장  
- [장면4] “AI 자동화, 모든 직무의 필수템” 자막  
- 엔딩: “No Code. No Stress. Just Flow.”

**BGM 아이디어:**  
- 트렌디한 Synthwave / EDM Drop 구간 강조  
- 리듬에 맞춰 Slack UI 애니메이션 연출
